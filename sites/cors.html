CORS = Cross-Origin Resource Sharing

CORS is a mechanism which gives a web application permission to access resources from a different origin.
This is achieved by using additional HTTP headers.

Example from MDN:

"An example of a cross-origin request: the front-end JavaScript code served from https://domain-a.com uses XMLHttpRequest
to make a request for https://domain-b.com/data.json."
Generally browser restrict cross-origion HTTP requests when they are initiated by a script.
This is simply a security measure.

CORS can cause major security risks such as leaking API key, other users data or worse, if configured wrongly.

Hence, CORS should only be implemented when absolutely neccessary. In addition, SOP is leveraged out by CORS.
